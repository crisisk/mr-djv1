Components: Performance Optimization

Richtlijnen voor het behalen van optimale Core Web Vitals scores en een snelle gebruikerservaring.

Core Web Vitals Doelstellingen
Largest Contentful Paint (LCP)
Doel: < 2.5 seconden

Meet de laadtijd van het grootste zichtbare element.

First Input Delay (FID) / INP
Doel: < 100 milliseconden

Meet de reactietijd op de eerste gebruikersinteractie.

Cumulative Layout Shift (CLS)
Doel: < 0.1

Meet de visuele stabiliteit van de pagina.

Optimalisatie StrategieÃ«n
  • **Afbeeldingen:** Gebruik moderne formaten (WebP, AVIF) en lazy loading voor off-screen beelden.
  • **Fonts:** Preload kritieke fonts (`Montserrat`) en gebruik `font-display: swap`.
  • **CSS/JS:** Minimaliseer en bundel assets. Gebruik tree-shaking en code splitting.
  • **Componenten:** Zorg voor minimale render-tijd. Vermijd zware JavaScript-frameworks waar mogelijk.
  • **Caching:** Implementeer sterke caching-headers voor statische assets.
  • **Server:** Gebruik een snelle CDN (zoals Netlify) voor wereldwijde distributie.
  • **CLS:** Reserveer ruimte voor afbeeldingen en embeds om layout shifts te voorkomen.
