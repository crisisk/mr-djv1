Documentation & Component Usage

Richtlijnen voor het gebruik van componenten, Storybook integratie en codevoorbeelden.

Component Documentatie
Storybook Integratie
Alle componenten zijn gedocumenteerd in Storybook. Dit is de primaire bron voor het bekijken van varianten, props en live voorbeelden.
Prop Tabellen
Elke component heeft een duidelijke tabel met alle beschikbare properties (props), hun types en standaardwaarden.
Do's & Don'ts
Specifieke richtlijnen over wanneer en hoe een component te gebruiken, en wat absoluut vermeden moet worden.
Developer Workflow
  • Importeer componenten via de centrale library.
  • Gebruik Design Tokens in plaats van hardcoded waarden.
  • Voer altijd een visuele regressietest uit na updates.
Code Voorbeeld: Button
// React Component Usage import { Button } from '@mrdj/eds-library'; const App = () => ( <Button variant="primary" size="large" onClick={() => console.log('Geboekt!')} > Boek Nu </Button> );
Design Tokens
Alle kleuren, spacing en typografie zijn gedefinieerd als tokens, waardoor ze eenvoudig in elk platform (Web, iOS, Android) kunnen worden gebruikt.
// Design Token Example (JSON) "color-primary-blue": { "value": "#00AEEF", "type": "color" }, "spacing-md": { "value": "16px", "type": "spacing" }
